Product Design Portofolio

Hello!, this article will cover DuitFusion design process.

DuitFusion is a concept for a mobile-based digital wallet transaction management application. The application is integrated with several e-wallets and mobile banking that are available in Indonesia.

Goals

This application is expected to help users manage their digital finances more quickly, making it easy to manage their finances effectively and easy to use for users who have multiple digital wallets and mobile banking. It combines an intuitive interface with innovative features to ensure that this application provides a user-friendly and enjoyable experience. DuitFusion will help users control their spending and transactions so that users financial can be more efficient.

Understand

"Financial Technology, or Fintech, is a term that refers to innovation in the financial services industry that incorporates modern technology." (Riyanto, Primiana, Yunizar, & Azis, 2018).

According to cnbcindonesia.com, the most common Fintech categories in Indonesia are payment (39%) and lending (24%).

    Based on the background, the following problems were found and summarized in the problem statement:

    1. There are a ton of digital financial provider in Indonesia like GoPay, OVO, Dana, LinkAja, etc
    2. It is difficult to manage digital wallet finances as users uses many applications.
    3. User may experience difficulty paying using QRIS when user has a not enough balance, so user have to choose a digital wallet with a sufficient balance or top up first.

    Research

    A well-defined problem statement helps to clarify the goals of this project. It identifies the specific problem that needs to be solved, and it describes the desired outcome. This information will be used to create scope, deliverable, and constrain that will help to achieve the user needs.

  • Scope
    1. Integration with digital wallets in Indonesia
    2. QRIS payment method
    3. history of balance and transactions.
  • Deliverable
    1. Features that can display balance and transaction history.
    2. QRIS payment feature that can choose a wallet account that has been integrated.
    3. Features that can categorize transactions.
  • Constrain
    1. The final product is a MVP.

    Analyze

    Moving out into analyzing user requirement, to identify potential users of the DuitFusion application, user identification is accomplished based on the activity of digital wallet users in Indonesia.

  • User identification

    • Potential users of the application are users who actively use multiple digital wallets, use QRIS payment methods as the main tool for transactions, and track transactions through transaction history.

  • Proposed Feature

    • The features developed in the DuitFusion application are as follows:

    1. Register
    2. Login
    3. PIN
    4. QRIS Payment
    5. Integrated digital wallet
    6. View balance
    7. View transaction history
    8. Categorized transaction
